跑数值造假指纹

GengScan 是一个 Claude Code / Agent 技能。它把一条一线观察到的朴素经验 —— 数值造假往往会在数据里留下指纹 —— 工程化成可复用流水线:从学者,到论文,到 Source Data,再到数据表里的算术指纹。

学术打假志愿者反复撞上同一堵墙:造假能查,但论文太多、人根本看不过来。GengScan 把上千篇论文压成几十个候选,让人把稀缺的判断力只花在数据本身已经显得反常的地方。机器从不宣布造假 —— 每一次命中都是待人核实的线索。

工作流程

从一份语料到一份分级候选清单:机器做分诊,人下判决。

语料

学者 → 论文 → 数据

把学者名单匹配到论文(OpenAlex 三重锚定),抓取它们的 Source Data 数据表。

→
检测

四类硬指纹

对每张表跑 A / G / F / H 检测器,每一类都对良性同形设了闸。

→
标记

异常与存疑

输出一份短候选清单 —— 确认的客观异常 / 存疑 / 良性,绝不是指控。

→
人工

人工复核

人开标记列、结合实验语境读,逐个给候选定性。

定性永远不是机器的活 —— 流水线只负责把可疑处规模化地摆到人面前。

四类硬指纹

每个检测器盯住一种造假的算术痕迹,并自带一个针对其合法「同形」的闸门。

A · 两列恒差 constant diff

两列之差恒为非零常数 —— 伪造时给一列统一加了偏移。

闸:排除绝对/相对派生值、坐标、堆叠绘图偏移 —— 看列头语义。
G · 块级小数锁定 decimal lock

一块区域小数位完全相同、整数不同 ——「保留小数、改整数」型伪造。

闸:粗网格数据天然重合;要求匹配小数 ≥6 种、主导占比 <60%。
F · 同图重复块 duplicate blocks

同一数值序列跨两列精确相同 —— panel 复制 / copy-paste 指纹。

闸:排除共享单调坐标轴与基线零列(≥40% 为 0)—— 它们是合法复用。
H · 行级重复 row duplicate

整行测量被复制到另一个样本/物种行 —— 跨样本 copy-paste。

闸:组学表行雷同是本质 —— 抑制批量重复,要求复制跨 ≥2 个不同样本标签。

扫描器还会计算更弱的信号(末位锁定、小数末位雷同、列内等差、单值碾压)。它们与良性机制重叠,因此只作保守线索,不进自动榜定论。

信号可靠性分层 — 诚实地读一次命中

每份报告都区分:哪些是事实、哪些悬而未决、哪些已核为干净。

确认的客观异常 · 可复现事实,性质仍待定 存疑 · 机器与人暂时都无法定性 良性 · 已开数据核实为派生 / 组学 / 噪声

跑些什么

四段可独立运行的环节 —— Python 脚本,可单独跑,也可交给 Claude 编排。

输入
一份学者名单(姓名、单位)或一组论文 DOI
GengScan 流水线
匹配与抓取

match_scholars · fetch_sources

OpenAlex 三重锚定 + ORCID + 机构门;代理优先抓取、解压、记录外链。

检测

forensic · image_forensic

A/G/F/H 数值指纹 + 良性闸;可选 ORB + RANSAC 图像取证。

机器分诊 人来定性 命中 ≠ 判决

内置一个合成阳性对照(植入 A/G/F/H 信号 + 一张干净负对照表),任何阈值改动后都能验证检测器既抓得到、又不误报。

核心特性

学者 → 论文匹配

三重锚定(姓名拼音 + 依托机构 + 研究领域)对接 OpenAlex,再做 ORCID 核身与逐篇机构门,对抗重名爆炸。

稳健的 Source Data 抓取

代理优先穿过反爬墙、解压 zip、记录 figshare/Zenodo/Dryad 外链 —— 并带一个 Dryad 工作量证明求解器。

四类带闸指纹

A 两列恒差、G 块级小数锁定、F 同图重复块、H 行级重复 —— 每类都对其合法同形设了闸。

组学假阳闸

自动识别并抑制组学/生信表里本质性的批量行重复,避免真正的几处 copy-paste 被淹没。

适用场景

适合

可规模化的第一遍分诊

  • 对一批论文(学者名单或 DOI)做第一遍筛查,把候选异常摆出来
  • 已有 Excel Source Data,想跑数值指纹
  • 复现或改造一套学术诚信筛查工具链
不适合

定性判决或文献检索

  • 给已确认的不端做法律定性
  • 据机器命中给实名学者扣造假帽
  • 文献检索,或没有可下载数据表的论文

快速开始

装进你的 Claude Code 技能目录

git clone https://github.com/xntj-ai/gengscan.git ~/.claude/skills/gengscan
cd ~/.claude/skills/gengscan && pip install -r requirements.txt

用合成阳性对照验证检测器

python scripts/make_synthetic.py
python scripts/forensic.py synthetic_positive.xlsx --out demo_report.txt

筛查一批 Source Data,再读候选

跑 python scripts/forensic.py --indir ./papers --out forensic_report.txt,或让 Claude 跑流水线并把确认的客观异常 / 存疑 / 良性分开报 —— 绝不是指控。

⚠ 免责声明

命中是待人核实的候选 —— 不是造假认定。

指纹命中可能有完全合理的解释(派生列、共享坐标轴、组学批量数据等)。任何指控都必须经过人工核实、作者解释与机构调查。本项目用于学术诚信的防御性研究与方法分享 —— 请负责任地使用,绝不据机器命中给实名学者扣造假帽。
