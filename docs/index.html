Research-integrity screening · The machine surfaces, a human decides

Screen scientific Source Data
for fabrication fingerprints

GengScan is a Claude Code / Agent skill. It turns one field-observed idea — numeric fabrication tends to leave fingerprints in the data — into a reusable pipeline: from scholars, to papers, to Source Data, to the arithmetic fingerprints inside the spreadsheets.

Integrity volunteers keep hitting the same wall: fabrication is detectable, but there are too many papers and too few eyes. GengScan compresses thousands of papers into a handful of candidates so a person spends their scarce judgment only where the data already looks odd. The machine never declares fabrication — every hit is a lead awaiting human verification.

How it works

From a corpus to a graded candidate list, the machine does triage and the human does the verdict.

Corpus

Scholars → papers → data

Resolve a scholar list to papers (OpenAlex triple anchoring) and fetch their Source Data spreadsheets.

→
Detect

Four hard fingerprints

Run the A / G / F / H detectors over every sheet, each gated against its benign look-alikes.

→
Flag

Anomaly & suspected

Output a short candidate list — confirmed objective anomaly vs. suspected vs. benign, never an accusation.

→
Human

Manual review

A person opens the flagged columns, reads them in experimental context, and characterizes each candidate.

Characterization is never the machine's job — the pipeline only puts the suspicious in front of a person, at scale.

The four hard fingerprints

Each detector targets one arithmetic trace of fabrication — and ships with a gate for its legitimate same-shape explanation.

A · Constant difference 两列恒差

Two columns differ by a fixed non-zero constant — a uniform offset added during fabrication.

Gate: exclude absolute-vs-relative derived values, coordinates, stacked-plot offsets — read the column header.
G · Decimal lock 块级小数锁定

A block keeps identical decimal parts while integer parts differ — "kept decimals, changed integers".

Gate: coarse-grid data coincides by nature; require ≥6 distinct decimals and dominant share <60%.
F · Duplicate blocks 同图重复块

An identical numeric sequence appears across two columns — a panel-copy / copy-paste fingerprint.

Gate: exclude shared monotonic axes and baseline-zero (≥40% zero) columns — they reuse legitimately.
H · Row-level duplicate 行级重复

A whole measurement row is copied to another sample / species row — cross-sample copy-paste.

Gate: omics tables repeat by nature — suppress bulk repeats, require the copy to span ≥2 distinct sample labels.

The scanner also computes softer signals (terminal-digit lock, decimal-tail repetition, in-column arithmetic, value domination). They overlap with benign mechanisms, so they stay conservative leads — not auto-list verdicts.

Reliability tiers — read a hit honestly

Every report separates what is fact, what is unresolved, and what was checked clean.

Confirmed objective anomaly · reproducible fact, intent still unsettled Suspected · neither machine nor human can characterize yet Benign · opened & verified as derived / omics / noise

What runs

Four standalone stages — Python scripts you can run on their own or let Claude orchestrate.

Inputs
A scholar list (name, org) or a set of paper DOIs
The GengScan pipeline
Resolve & fetch

match_scholars · fetch_sources

OpenAlex triple anchoring + ORCID + institution gate; proxy-first fetch, zip, external links.

Detect

forensic · image_forensic

A/G/F/H numeric fingerprints with benign gates; optional ORB + RANSAC image forensics.

Machine triages Human characterizes A hit ≠ a verdict

A built-in synthetic positive control (planted A/G/F/H signals + one clean negative-control sheet) lets you verify the detectors fire — and don't false-trigger — after any threshold change.

Core features

Scholar → paper resolution

Triple anchoring (pinyin name + institution + field) against OpenAlex, then ORCID identity check and a per-paper institution gate to fight name collisions.

Robust Source Data fetching

Proxy-first through anti-scraping walls, zip extraction, and recorded figshare / Zenodo / Dryad links — with a Dryad proof-of-work solver.

Four gated fingerprints

A constant difference, G decimal lock, F duplicate blocks, H row duplicates — each gated against its legitimate same-shape explanation.

Omics false-positive gate

Bulk row duplication typical of omics / bioinformatics tables is identified and suppressed, so a few real copy-pastes are not buried.

When to use

Good fit

Scalable first-pass triage

  • A first pass over many papers (scholar list or DOIs) to surface candidate anomalies
  • You already have Excel Source Data and want to run the numeric fingerprints
  • Reproducing or adapting an integrity-screening toolchain
Not for

Verdicts or search

  • Legally characterizing already-confirmed misconduct
  • Pinning a fabrication label on a named scholar from a machine hit
  • Literature search, or papers with no downloadable data tables

Quick start

Install into your Claude Code skills directory

git clone https://github.com/xntj-ai/gengscan.git ~/.claude/skills/gengscan
cd ~/.claude/skills/gengscan && pip install -r requirements.txt

Verify the detectors with a synthetic positive control

python scripts/make_synthetic.py
python scripts/forensic.py synthetic_positive.xlsx --out demo_report.txt

Screen a directory of Source Data, then read the candidates

Run python scripts/forensic.py --indir ./papers --out forensic_report.txt, or ask Claude to run the pipeline and report confirmed anomaly vs. suspected vs. benign — separately, never as an accusation.

⚠ Disclaimer

A hit is a candidate awaiting human verification — not a finding of fabrication.

A fingerprint match can have a perfectly legitimate explanation (derived columns, shared axes, bulk omics data, and more). Any allegation must go through human verification, the authors' explanation, and an institutional investigation. This project is for defensive research-integrity work and method sharing — use it responsibly, and never use a machine hit to pin a fabrication label on a named scholar.
