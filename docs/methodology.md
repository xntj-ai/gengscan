# 方法论 / Methodology

GengScan 的设计哲学:**机器把上千篇论文压成几十个候选,人把候选定性。** 工具的价值不在「判定造假」(机器做不到),而在「规模化地把可疑处摆到人面前」。

---

## 1. 学者 → 论文:三重锚定

把一份中文学者名单匹配到英文文献库,核心难点是**中文名进 OpenAlex 只剩拼音、重名爆炸**(「Wang Wei」「Li Kai」成百上千)。

**匹配 = 拼音 + 依托机构 + 研究领域**(三者同时命中):
- 姓 + 名拼音严格匹配(防止抓到名气大的同名同事)
- 机构门:候选作者的 OpenAlex institution id 命中依托单位
- 领域门:院系关键词映射到 OpenAlex topics.field,过滤同名不同领域者

**两道校验:**
- **ORCID 核身** — 取匹配作者的 orcid,查官方登记名核对,名不符剔除
- **逐篇机构门** — 对较可信者,只采信「该作者在依托单位发表」的论文 DOI,其余进存疑

唯一同名 = 高可信;同领域多同名取引用最高 = 较可信;实在分不开 = 单列「需人工核对」,不硬塞。

## 2. 四类硬指纹:判据与反例

造假的数字不像真实测量那样自然。但**每一类指纹都有合法的「同形」解释**,这正是必须人看的原因。

### A · 两列恒差
两列数值之差恒为常数。**造假**:伪造时给一列统一加了偏移。
**反例(必排除)**:绝对能量 vs 相对能量(差 = 参考态能量)、基因组 start/end 坐标(差 = 1bp)、多曲线堆叠绘图(差 = 视觉偏移)、背景/空白扣除、峰拟合派生。→ 看列头语义。

### G · 块级小数锁定
一块区域小数位完全相同、整数不同。**造假**:「保留小数部分、改整数部分」型伪造。
**反例**:粗网格数据(如全 .0/.5 的光谱峰位)两列错位对齐时,小数天然高频重合。
**闸**:匹配的小数须足够多样(≥6 种且主导 <60%),否则判粗网格假阳。

### F · 同图重复块
同一图内整段数值序列跨列精确相同。**造假**:panel 复制。
**反例**:共享单调坐标轴(每条曲线配同一 x 列)、基线零列(虚部起点)、共享拟合曲线。
**闸**:排除单调列(共享轴)与 ≥40% 为 0 的列(基线)。

### H · 行级重复
整行测量被复制到另一个样本行。**造假**:跨样本 copy-paste(把一个样本的数据复制给另一个)。
**反例(最重要)**:组学/生信表(微生物组丰度、radiomics 特征、系统发育树节点、测序 read ID、基因符号)里,不同行数值雷同是**计算管线批量产出的本质**,不是造假。
**闸**:单 sheet 行级重复命中过多 → 判组学批量数据,整体抑制。校准时确保真实的小型手工录入实验表(跨不同样本标签的整行复制)仍保留。

## 3. 信号可靠性分层

- **硬指纹(A 恒差 / G 小数锁定 / F 同图重复 / H 行复制)** — 能筛出候选,但**无一能由机器直接定性**。
- **软信号(末位分布 / Benford / 小数雷同)** — 与良性机制(取均值、四舍五入、图上数字化)同形,**必须人结合实验语境判**,不进自动榜。

---

## 4. 图像取证(可选模块)

数值工具碰不到图像造假。图像模块用 **ORB 局部特征匹配**(而非感知哈希 pHash):
- pHash 抓整图近重复,**漏局部拼接 / 旋转翻转复用**(western blot 条带复制)。
- ORB 提取局部关键点,BF 匹配 + Lowe ratio + RANSAC 几何验证,能定位「图 A 的一块 = 图 B 的一块」,且抗旋转缩放。

**已知难点(假阳控制):** 科学图有大量通用元素(坐标轴、字母标注、流程示意图、分子结构),ORB 会把这些当匹配。需要:① 占比阈值(真复制是高占比连续块) ② 排「明星图」(和很多篇都匹配的图 = 通用图元) ③ 聚焦 western blot/凝胶/显微图,排除示意图/光谱 ④ 候选交人(或视觉大模型)逐对终审。

> 图像取证假阳控制是硬骨头,需迭代。当前为已验证可行的 MVP,不像数值线那样直接出结论。

## 5. 实战踩过的坑

- **抓取层比检测层更难。** 目标站反爬墙(直连 0 字节,需代理);Source Data 不止 .xlsx(常打包成 zip,或散在 figshare/Zenodo/Dryad);区分「抓取失败」与「真无数据」——否则覆盖率被高估。
- **超大表会挂死解析器。** 100–190MB 的原始矩阵让 openpyxl 卡死。防御:文件大小门 + subprocess 单文件超时 + 逐目录进度打点。
- **覆盖率要按「实际喂进检测器的」算,不是「跑过的篇数」。** 一篇没抓到数据 ≠ 这篇干净。
- **每次改阈值都要跑合成阳性对照回归。** 校准集恰好不含某种良性形态时,会误以为「零假阳」;真实多样数据才暴露假阳模式。

---

## 6. 一条贯穿始终的纪律

**绝不据筛查命中给实名学者扣造假帽。** 工具的输出是「待人核实的候选」。报告时严格分级:

| 级别 | 含义 |
|------|------|
| 确认的客观异常 | 可复现的事实(如两行不同量纲指标逐位全同),但性质(无意错误 vs 蓄意)仍需作者解释 |
| 存疑 | 工具与人工均无法定性,需图注 / 领域专家 |
| 良性 | 已开数据核实为派生列 / 组学 / 噪声 |

定性是人的工作,机器永远只提供线索。
